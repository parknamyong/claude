# AI 음악교실

초등학생 대상 AI 음악 교육 프로그램을 소개하고, 수강 신청 및 관리가 가능한 웹사이트 프로토타입

## 프로젝트 정체성

| 항목 | 내용 |
|------|------|
| 서비스명 | AI 음악교실 |
| 목적 | 초등학생 대상 AI 음악 교육 프로그램 소개 + 수강 신청(결제 UI 포함) + 관리 |
| 성격 | **실제 운영 서비스가 아닌, 완전히 작동하는 React 기반 프로토타입** |

## 기준 문서

- **유일한 기준**: `AI 음악교실 웹사이트 개발 – 프로젝트 브리핑 PDF`
- PDF에 명시된 내용은 추측 없이 그대로 반영
- 문서에 없는 내용은 임의로 추가하지 않음

## 핵심 기능 범위

### 1. 고객용 페이지
- 홈
- 프로그램 상세 (기초/보컬/연주)
- 소개
- 수강 신청
- 후기
- 공지사항
- 갤러리
- 오시는 길

### 2. 수강 신청
- 즉시 결제 (UI만)
- 상담 후 결제 플로우
- 정원 관리, 모집중/마감 표시
- 대기자 접수

### 3. 학부모 대시보드
- 자녀 수강 정보
- 일정
- 출석
- 결제 내역
- 공지 확인

### 4. 어드민 대시보드
- 학생/프로그램/반/출석/결제/환불/공지/갤러리 관리
- 통계 요약 화면

## 기술 제약 (매우 중요)

| 허용 | 금지 |
|------|------|
| React 단일 애플리케이션 | 서버, DB, API, 백엔드 구현 |
| React useState | localStorage / sessionStorage |
| 결제 UI/플로우 시뮬레이션 | 실제 결제 연동 |
| Placeholder 이미지/영상 | Persistent Storage |

## 디자인 기준

- 브랜드 컬러, UI 구조, 페이지 구성: **PDF 기준**
- 반응형 필수
- 애니메이션은 과하지 않게
- 이미지/영상은 placeholder로 처리

## 작업 규칙

1. **PR 단위 작업**: 모든 작업은 PR 단위로 진행
2. **기능 = PR**: 기능 하나 = PR 하나
3. **Plan 필수**: 변경 전 반드시 Plan 제시
4. **추측 금지**: 불확실하면 질문 또는 확인 방법 제시

### 커밋 컨벤션

```
<type>(<scope>): <subject>

예시:
feat(enrollment): 수강 신청 폼 구현
fix(dashboard): 출석 현황 표시 오류 수정
docs(readme): 프로젝트 범위 업데이트
```

**Type 종류**:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 코드 포맷팅
- `refactor`: 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드, 설정 변경
