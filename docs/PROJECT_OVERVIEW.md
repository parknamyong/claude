# 시스템 개요

## 프로젝트 정체성

| 항목 | 내용 |
|------|------|
| 서비스명 | AI 음악교실 |
| 목적 | 초등학생 대상 AI 음악 교육 프로그램 소개 + 수강 신청(결제 UI 포함) + 관리 |
| 성격 | **완전히 작동하는 React 기반 프로토타입** (실제 운영 서비스 아님) |

## 기준 문서

- **유일한 기준**: `AI 음악교실 웹사이트 개발 – 프로젝트 브리핑 PDF`
- PDF에 명시된 내용은 추측 없이 그대로 반영
- 문서에 없는 내용은 임의로 추가하지 않음

## 수강 신청 플로우

### 즉시 결제

```
프로그램 선택 → 정보 입력 → 결제 UI → 완료 화면
                              │
                              └── (UI 시뮬레이션만, 실제 결제 없음)
```

### 상담 후 결제

```
프로그램 선택 → 상담 신청 → 상담 완료 → 결제 안내
```

### 정원 관리

```
┌──────────────────────────────────────┐
│  프로그램 정원 상태                    │
├──────────────────────────────────────┤
│  정원 여유 있음  →  "모집중" 표시      │
│  정원 마감      →  "마감" 표시        │
│                    └── 대기자 신청 가능│
└──────────────────────────────────────┘
```

## 로그인 시스템 (UI 시뮬레이션)

> **중요**: 실제 인증 없음. 목업 계정으로 UI 플로우만 시뮬레이션

### 로그인 플로우

```
로그인 폼 → useState로 상태 관리 → 대시보드 접근
            (목업 계정 매칭)
```

### 목업 계정 (예시)

| 유형 | 이메일 | 비밀번호 |
|------|--------|----------|
| 학부모 | parent@test.com | 1234 |
| 어드민 | admin@test.com | admin |

## 결제 시스템 (UI 시뮬레이션)

> **중요**: 실제 결제 연동 없음. UI 플로우만 구현

```
결제 수단 선택 → 결제 정보 입력 → "결제" 버튼 클릭 → 완료 화면
                                        │
                                        └── (실제 결제 처리 없음)
```

## 데이터 관리

### 목업 데이터

모든 데이터는 상수(constants)로 정의하여 사용

```
src/
└── constants/
    ├── programs.ts    # 프로그램 목업 데이터
    ├── students.ts    # 학생 목업 데이터
    ├── reviews.ts     # 후기 목업 데이터
    ├── notices.ts     # 공지사항 목업 데이터
    └── gallery.ts     # 갤러리 목업 데이터
```

### 상태 관리

- **페이지 새로고침 시 초기화됨** (영구 저장소 없음)
- 모든 상태는 `useState`로 관리
- 앱 최상위에서 필요시 상태 공유 (props drilling 또는 Context)
